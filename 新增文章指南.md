# 新增文章完全指南

## 一、项目目标

这是一个 AI 工具选型 SEO 内容站（域名 `suiliuxiaomi.com`），通过搜索流量引导用户进入小店（`pay.ldxp.cn/shop/ai007`）完成购买转化。

**你唯一需要做的事**：选题 → 写文章 → `git push`。推送后全自动完成：构建 → SEO 检查 → Cloudflare Pages 部署 → 通知搜索引擎（IndexNow + 百度 + sitemap ping）。

---

## 二、新建文章完整流程

### 第 1 步：确定文件名

在 `src/content/posts/` 下新建 Markdown 文件。文件名即 URL slug，规则：

- 使用英文 + 连字符（kebab-case）
- 简短、含核心关键词
- 示例：`claude-pro-vs-chatgpt-plus.md`、`ai-tools-for-teachers.md`

### 第 2 步：填写 Frontmatter

复制下面的模板，填入你的内容。**所有字段必填。**

```yaml
---
title: "文章标题（必须包含核心搜索关键词）"
description: "文章描述（120-160字，包含核心关键词和变体）"
slug: "your-article-slug"
category: "工具选择"
primaryKeyword: "用户会搜索的核心词"
secondaryKeywords:
  - "长尾搜索词1"
  - "长尾搜索词2"
  - "长尾搜索词3"
intent: "scenario"
updatedAt: "2026-06-05"
ctaText: "进入AI工具小店选购"
faq:
  - question: "FAQ问题1（第一个问题最好包含主关键词）？"
    answer: "答案内容"
  - question: "FAQ问题2？"
    answer: "答案内容"
  - question: "FAQ问题3？"
    answer: "答案内容"
  - question: "FAQ问题4？"
    answer: "答案内容"
  - question: "FAQ问题5？"
    answer: "答案内容"
---
```

#### Frontmatter 字段约束速查

| 字段 | 约束 | 说明 |
|------|------|------|
| `title` | 任意字符串 | 页面的 `<title>` 和 `<h1>`，含核心关键词 |
| `description` | 任意字符串 | 页面的 `<meta description>`，搜索摘要展示 |
| `slug` | 必须与文件名一致 | URL 路径，不含 `/` |
| `category` | 三个值之一 | `"工具选择"` / `"工具对比"` / `"场景需求"` |
| `primaryKeyword` | 任意字符串 | 文章目标搜索词 |
| `secondaryKeywords` | 数组，至少 1 个 | 辅助搜索词 |
| `intent` | 四个值之一 | `"tool"` / `"comparison"` / `"scenario"` / `"guide"` |
| `updatedAt` | `YYYY-MM-DD` 格式 | 文章更新日期，同步到 sitemap lastmod |
| `ctaText` | 任意字符串 | CTA 按钮文案 |
| `faq` | 恰好 **5** 个问答 | 少于或多于 5 个都会构建失败 |

### 第 3 步：写正文

正文结构自由，但必须满足以下硬性指标（第 4 步构建时会自动检查）。

---

## 三、正文硬性指标（构建时自动校验，不达标 = 构建失败）

### 3.1 内容量

| 指标 | 最低要求 | 不达标后果 |
|------|----------|-----------|
| 中文字数 | **≥ 1200 字** | 构建失败 |
| CTA 购买链接 | **≥ 3 个** | 构建失败 |
| 站内链接（href 以 `/` 开头） | **≥ 5 个** | 构建失败 |
| H1 标题 | **恰好 1 个**（通常即文章标题） | 构建失败 |
| H1 唯一性 | 不能与其他文章 **重复** | 构建失败 |

### 3.2 CTA 购买链接写法

CTA 按钮必须使用以下格式之一。推荐使用 Markdown 链接语法（更简单）：

**方式 A：Markdown 链接（推荐）**

```markdown
[进入AI工具小店选购](https://pay.ldxp.cn/shop/ai007?utm_source=seo&utm_medium=organic&utm_campaign=ai_tools&utm_content=你的slug)
```

必须包含的属性：
- `target="_blank"` → 但 Markdown 链接不支持此属性，可直接写完整 URL
- 链接必须带 UTM 参数，`utm_content` 填当前文章 slug

**方式 B：HTML（需要 target 等属性时）**

```html
<a href="https://pay.ldxp.cn/shop/ai007?utm_source=seo&utm_medium=organic&utm_campaign=ai_tools&utm_content=你的slug"
   target="_blank"
   rel="nofollow sponsored noopener"
   data-shop-cta
   data-page-slug="你的slug"
   data-cta-position="top">
   进入AI工具小店选购
</a>
```

> **注意**：`data-shop-cta` 属性必须有，SEO 检查脚本据此统计 CTA 数量。建议在文章开头、中间、结尾各放一个，确保 ≥ 3 个。

### 3.2.1 CTA 文案与位置策略（影响转化率的关键）

**文案必须场景化，不要所有文章千篇一律用"进入AI工具小店选购"。**

| 文章类型 | 推荐 CTA 文案示例 |
|----------|-------------------|
| 工具选择（ChatGPT/Claude 等） | "查看 ChatGPT Plus 当前可选方案" |
| 工具对比 | "按你的用途进入小店选择" |
| 编程场景 | "查看适合编程的 AI 工具方案" |
| 论文/写作场景 | "查看写作类 AI 工具方案" |
| 学生党/低成本 | "查看学生党可用方案" |
| 通用导购 | "前往小店按用途查看分类" |

**修改方式**：直接改 frontmatter 的 `ctaText` 字段，模板自动读取。正文中嵌入的购买链接文案同理，按场景改写。

**位置策略**：不要只在文章开头和结尾放 CTA。最佳位置是**每个关键判断节点之后**：

```markdown
## 场景推荐表

| 使用场景 | 推荐方向 | 购买前提醒 |
| ... | ... | ... |

[按场景进入小店查看对应方案](https://pay.ldxp.cn/...)

## 核心判断：先看用途

...正文内容...

[已经明确用途？进入小店按分类选择](https://pay.ldxp.cn/...)

## 最后总结

...总结内容...

[确认需求，进入小店查看当前可选方案](https://pay.ldxp.cn/...)
```

> **原则**：用户读完一段"为什么"之后，立刻给一个"去哪买"的入口。CTA 跟着判断结论走，转化率远高于只在模板固定位置放三个。

### 3.3 站内链接要求

正文中必须至少有 **5 个**指向站内其他页面的链接。所有站内链接以 `/` 开头。

## 五、绝对禁止项（触发构建失败或违规）

### 5.1 禁止使用的词语

文章中**绝对不能出现**以下词语（SEO 检查脚本会扫描）：

```
破解、绕过、共享账号、低价卡密、卡密秒发、
不封号、保号、灰产、平台不让发、薅羊毛
```

### 5.2 禁止的内容行为

- 不展示虚假价格、库存、评分、评价
- 不使用 Product/Offer/Review 结构化数据
- 不复制小店商品全集
- 不做自动跳转、隐藏文字、关键词堆砌
- 不把 AI 输出当最终答案
- 所有购买入口必须是用户主动点击的链接
- Shop 链接必须使用 `rel="nofollow sponsored noopener"`

## 六、可灵活变更的部分

以下内容**没有硬性要求**，根据选题自由发挥：

- 正文结构（章节、标题层级、段落顺序）
- 是否使用表格、列表、引用块
- 是否插入图片（如有图，必须写 `alt` 属性）
- 文章长度（只要 ≥ 1200 中文字）
- CTA 文案（`ctaText` 字段可自定义）
- 写作风格和语气
- 内链选择（从速查表选 5 个以上即可）

## 七、推送前自检清单

写完文章后，对照检查：

- [ ] 文件名：`src/content/posts/xxx.md`，英文 kebab-case
- [ ] Frontmatter：10 个字段全部填写
- [ ] `faq`：恰好 5 个问答，无遗漏
- [ ] `updatedAt`：日期格式 `YYYY-MM-DD`
- [ ] 中文字数：≥ 1200
- [ ] CTA 链接：≥ 3 个（每个都带 `data-shop-cta` 属性）
- [ ] 站内链接：≥ 5 个（`href` 以 `/` 开头）
- [ ] 敏感词：确认无禁止词语
- [ ] 图片 alt：如有图片，均已填写
- [ ] H1：只有 1 个一级标题
- [ ] 本地验证：运行 `npm run build`，确保输出 `SEO check passed`

### 本地验证命令

```bash
npm run build
```

看到以下输出即表示通过：

```
SEO check passed for XX pages.
```

如果失败，根据错误提示修改后重新运行，直到通过。

## 九、常见错误

| 错误信息 | 原因 | 解决 |
|----------|------|------|
| `expected one H1, found N` | 文章中 H1（`#`）超过 1 个 | 正文改用 `##` 开头 |
| `fewer than 1200 Chinese chars` | 中文字数不够 | 扩充正文内容 |
| `fewer than 3 CTA links` | CTA 购买链接不够 | 多放几个购买入口链接 |
| `fewer than 5 internal links` | 站内链接不够 | 从速查表中多选几个相关链接 |
| `forbidden term XXX` | 正文含禁止词语 | 删除或替换该词 |
| `missing title/description/canonical` | Frontmatter 字段缺失 | 检查 frontmatter 是否完整 |
| `duplicate H1 with ...` | 标题与已有文章重复 | 修改 `title` 字段 |
| `sitemap missing /xxx/` | 页面未出现在 sitemap | 检查 slug 和文件名是否一致 |
| `broken internal link /xxx/` | 站内链接指向不存在的页面 | 从速查表中核对链接路径 |
